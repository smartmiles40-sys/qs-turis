   mensagem copiada). A rota `/api/chatapp-send` agora aceita o login do próprio

     com fallback automático pro WhatsApp se o webfone não tiver token.

   - Ligação SIP (softphone) segue disponível quando ativada em Configurações.

## Como funciona por dentro (referência)

```
Bitrix (negócio novo) ──webhook──▶ n8n ──POST /api/lead-inbound──▶ QS
                                                                    │ bitrix_id
QS (perdido/ganho/reunião/nota) ◀──polling 1min── n8n ──crm.deal.update /
                                                        crm.timeline.comment.add──▶ Bitrix

ChatApp: n8n (6/6h) ──token──▶ qs_settings ──▶ /api/chatapp-send ──▶ lead
```

- Flags de sync: `qs_leads.bitrix_status_synced`, `qs_meetings.bitrix_synced`,
  `qs_notes.bitrix_synced` (histórico antigo entra como sincronizado — o Bitrix
  não é inundado com passado).
- Leads antigos sem `bitrix_id` não movem coluna (não tem como saber qual card
  é); ficam com `bitrix_error` marcado. Todo lead novo já nasce vinculado.
- Detalhes de cada workflow: `n8n/README.md`.
